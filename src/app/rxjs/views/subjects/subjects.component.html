Subjects

What is Subject in RxJS?

It's a special type of Observable that can also act as an Observer. This means it can both emit data (like an Observable) and receive data (like an Observer).

Consider a newsletter service. The newsletter service is like a Subject. It can send out newsletters (emit data) to all its subscribers. Subscribers (Observers) can receive these newsletters.

Now, let's say the newsletter service allows its subscribers to contribute articles. Subscribers can send their articles to the newsletter service (Subject), and the service can then include these articles in the next newsletter and send it out to all subscribers.

In this analogy, the newsletter service (Subject) is both emitting data (sending newsletters) and receiving data (receiving articles from subscribers). This is similar to how a Subject in RxJS can both emit data to its subscribers and receive data from them.

Subject vs Observable

A regular Observable can only emit data but not receive it. Subject is capable of both emitting data and also receiving the data.

Types of Subject

In RxJS, there are 4 types of subjects:

  1. Subject: This is the basic type. It allows values to be multicasted to many Observers. When a value is emitted, it will be passed to all the subscribers.
  2. ReplySubject: This variant of Subject records multiple values from the Observable execution and replays them to new subscribers.
  3. BehaviorSubject: This is a variant of Subject that requires an initial value and emits its current value (last emitted item) to new subscribers.
  4. AsyncSubject: This is another variant of Subject, that emits the last value (and only the last value) emitted by the source Observable, and only after that source Observable completes.
